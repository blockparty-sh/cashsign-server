CashSign

Cash Contract signing protocol

Created by slp.cash developers James Cramer and Jt Freeman.

Trusted services suck

Building decentralized applications is difficult and error prone, requiring lots of testing. How would Bitcoin look today if it was easier than building centralized trusted applications?

Stop writing single-app wallets

During building of craft.cash I had to write my own wallet to make an app. That took half the time. Quality wallets already exist and their testing and security should be inherited by your apps, you are not competing with Electron Cash or Electrum.

Sign arbitrary transactions using your existing wallet

Websites can now use a URI which provides a message or a partially signed transaction to a CashSign enabled wallet, enabling trustless smart contracts, crowd sales, token swaps, login systems, and more.

How does it work?

A webapp provides URIs that your wallet can consume

For message signing (can be used for authentication systems)


simpledger:qzyp3sqdfdry8q3d67n3hjluc3r7km47xq28smxth4
?cashsign-type=message
&cashsign-data=0x68656c6c6f20776f726c64
&cashsign-callback-url=http://localhost:5000/cashsign-callback

What we built

  • An extension to the simpleledger URI scheme to support signing messages and transactions.
  • A fork of Electron Cash SLP Edition (Electrum Wallet fork on BCH) which supports the new URI scheme
  • A web app which demos a simple signing of messages
  • A SLP token called vires
  • A trustless crowdsale of vires that demonstrates the possibilities for partially signed transactions

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
