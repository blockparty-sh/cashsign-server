CashSign

A signing protocol for trustless token crowdsales, and more!

Created by slp.cash developers on Bitcoin Cash.

Trusted services suck

Building decentralized apps is difficult requiring lots of testing. How would Bitcoin look today if it was easier than building centralized trusted applications?

Stop writing single-app wallets

Quality wallets already exist and their testing and security should be inherited by your apps, you are not competing with Electron Cash or Electrum.

Your time should be better spent making your application, and your users will prefer a unified experience.

Sign arbitrary transactions using your existing wallet

Websites can now use a URI (Uniform Resource Identifier, like a url) which provides a message or a partially signed transaction to a CashSign enabled wallet, enabling trustless smart contracts, crowd sales, token swaps, login systems, and more.

What we built

  • A URI scheme to support bitcoin signing using wallets.
  • A demo implementation of the URI scheme, in Electron Cash SLP Edition
  • A web app to demonstrate signing of messages
  • A SLP token called vires
  • A trustless crowdsale of vires to demonstrate interaction with partially signed transactions

Demo time pt.1

First let's enter an address and sign the phrase "hello world" using our wallet @ Prove Yourself!.

The website will store the wallets signature in a database which we can view at the Admin page

Demo time pt.2

Now let us trustlessly exchange BCH for SLP tokens at our Crowd sale!.

These transactions have been created with a SLP input of 21 vires tokens which is signed, and an unsigned input of bch from you - the user. The outputs will be 21 vires tokens to you, and 100K sats to the crowdsale operator.

Clicking buy will allow you to inspect and sign the transaction.

How does it work?

A webapp provides URIs that your wallet can consume

For message signing (can be used for authentication systems)


simpledger:qzyp3sqdfdry8q3d67n3hjluc3r7km47xq28smxth4
?cashsign-type=utf8
&cashsign-data=0x68656c6c6f20776f726c64
&cashsign-callback-url=http://localhost:5000/cashsign-callback

For transaction signing (token swaps, crowdsales, etc)


simpledger:qzyp3sqdfdry8q3d67n3hjluc3r7km47xq28smxth4
?cashsign-type=txn
&cashsign-data=0x[FULL TX HEX HERE]
&cashsign-callback-url=http://localhost:5000/crowdsale-callback

What this enables

  • p2sh smart contracts can now be spent without requiring developers to also design a custom wallet
  • token platforms can now be focused on the negotiation between parties and the construction of transactions
  • cross platform sign-up/login systems (think MetaMask's one-click login)
  • a large reduction in development time required for dApps
  • a safer and more unified user experience
